# Codex Quiz コンテンツ品質基準

Claude Code Quizの運用で確立された品質基準をCodex向けに翻訳する。事実の一次情報はOpenAI公式Codexマニュアルとし、Claude固有の仕様は流用しない。

## 評価する3軸

- `difficulty`: beginner / intermediate / advanced。理解の複雑さ。
- `value`: practical / neutral / trivia。実務で使う頻度と効果。
- `topic`: 公式マニュアルのページ・節との対応。

advancedでも日常的な並列作業ならpracticalになり得る。beginnerでも稀なフラグ名の暗記だけならtriviaになり得る。難易度と価値を混同しない。

## 採用基準

- 機能を使う場面や、複数機能の使い分けを判断できる。
- よくある失敗、危険な操作、設定ミスを避けられる。
- 公式ドキュメントの重要な推奨ワークフローを再現する。
- 単独の名称暗記より、目的・状況・結果を問う。
- 既存問題と同じ事実の言い換えだけになっていない。

## 目標データ

- 3桁の一意なID、カテゴリ、difficulty、value、topic
- 明確な問題文と、もっともらしい4選択肢
- 不正解ごとのwrongFeedback（16文字以上を目安）
- 背景と使いどころを含む解説
- OpenAI公式のreferenceUrlと可能なら節アンカー
- verifiedAt（最終事実確認日）

## 機械検査

- ID、カテゴリ、正解インデックス、選択肢数
- 問題文・選択肢・解説の重複
- 正解位置が1箇所へ35%以上偏らない
- 正解だけが極端に長い、またはコード書式を持つ状態を検出
- 短すぎる、冗談と分かる、文法上不自然な誤答を検出
- 不正解ごとのwrongFeedbackを必須化
- 公式URLのドメイン、ページ、アンカーを検証
- practicalとtriviaの同時付与を禁止
- 各難易度が全体の10%以上あるか監視

表示時には選択肢をランダム化し、正解位置の記憶で解けないようにする。

## Verified Factsとdrift監査

CLI、設定値、成熟度、対応surface、既定動作など更新されやすい事実をVerified Factsへ集約する。公式マニュアル更新時はtopicから影響する問題IDを逆引きし、正解そのものを再検証する。古い集約資料より現在の個別公式ページを優先する。

## 実務価値の利用

practical問題は通常出題とSRS期限超過問題の同条件時に弱く優先する。価値スコアは難易度の代用にせずtie-breakとして使う。triviaは上級者検索や実力テストには残せるが、初心者の学習パスでは優先しない。

## 現在の移行課題

1. `wrongFeedback`を全問題へ段階的に付与する（134 / 240問。データ形式・画面・検査は実装済み）。
2. ページ名だけの`source`を公式referenceUrlへ移行する。
3. 全問題へdifficulty、value、topic、verifiedAtを付与する。
4. CLI名称暗記問題を実務シナリオ型へ書き換える。
5. 選択肢品質lintと公式URL検査をCIへ追加する。
6. OpenAI公式マニュアルのカバレッジ表とVerified Factsを整備する。
