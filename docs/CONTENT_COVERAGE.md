# 公式ドキュメント・カバレッジ

| 領域 | 実務価値 | 状態 | 次の重点 |
|---|---:|---|---|
| Prompting・完了条件 | 最重要 | 基礎あり | 大規模変更と失敗時の依頼 |
| AGENTS.md | 最重要 | 概ね対応 | monorepo実例 |
| 承認・sandbox | 最重要 | 基礎あり | networkと権限エラー |
| Review | 最重要 | 着手 | PR指摘修正ループ |
| Session・context | 高 | 着手 | archive、compactの実例 |
| Worktree | 高 | 着手 | branch制約、cleanup |
| App・IDE・CLI・Cloud | 高 | 着手 | surface別の操作差 |
| Skills・MCP | 高 | 基礎あり | 認証とtool metadata |
| Hooks・Plugins | 中〜高 | 入口のみ | trustとmanifest |
| Automations | 中〜高 | 未対応 | scheduled task |
| Troubleshooting | 中 | doctorのみ | ログ、version、worktree |

「着手」は、公式URL・topic・value・verifiedAtを持つ問題が存在する状態。問題数だけで完了とせず、主要な判断と失敗回避を説明できるまで追加する。
